Use the Spaces API to list available spaces, create new ones, retrieve full space details, and inspect space membership — all programmatically. Spaces are the primary organizational unit in Company Brain: each space groups related knowledge items together and carries its own set of permissions, making it easy to scope content to a specific team, project, or topic area.

## GET /v1/spaces — List all spaces

Returns an array of all spaces the authenticated API key has access to. This includes spaces where the associated user is an owner, admin, editor, or viewer.

### Example request

```bash
curl "https://api.companybrain.ai/v1/spaces" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example response

```json
{
  "data": [
    {
      "id": "sp_01hx",
      "name": "People & HR",
      "description": "Policies, onboarding guides, and HR resources for all employees.",
      "slug": "people-hr",
      "color": "#4F46E5",
      "icon": "users",
      "member_count": 34,
      "item_count": 112
    },
    {
      "id": "sp_02yz",
      "name": "Product",
      "description": "Roadmaps, specs, and release notes for the product team.",
      "slug": "product",
      "color": "#059669",
      "icon": "rocket",
      "member_count": 18,
      "item_count": 87
    }
  ]
}
```

### Response fields

<ResponseField name="data" type="array">
  An array of space objects.

  <Expandable title="Space fields">
    <ResponseField name="id" type="string">
      The unique identifier for the space.
    </ResponseField>
    <ResponseField name="name" type="string">
      The display name of the space.
    </ResponseField>
    <ResponseField name="description" type="string">
      A short description of the space's purpose or content.
    </ResponseField>
    <ResponseField name="slug" type="string">
      The URL-friendly slug used in the Company Brain web app (e.g. `people-hr`).
    </ResponseField>
    <ResponseField name="color" type="string">
      The hex color code assigned to the space (e.g. `#4F46E5`).
    </ResponseField>
    <ResponseField name="icon" type="string">
      The Font Awesome icon name assigned to the space (e.g. `users`, `rocket`).
    </ResponseField>
    <ResponseField name="member_count" type="integer">
      The number of users who are members of this space.
    </ResponseField>
    <ResponseField name="item_count" type="integer">
      The number of knowledge items published in this space.
    </ResponseField>
  </Expandable>
</ResponseField>

## GET /v1/spaces/:id — Get a single space

Retrieves full details for a single space by its ID, including timestamps that are omitted from the list response.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the space to retrieve.
</ParamField>

### Example request

```bash
curl "https://api.companybrain.ai/v1/spaces/sp_01hx" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example response

```json
{
  "id": "sp_01hx",
  "name": "People & HR",
  "description": "Policies, onboarding guides, and HR resources for all employees.",
  "slug": "people-hr",
  "color": "#4F46E5",
  "icon": "users",
  "created_at": "2023-09-01T08:00:00Z",
  "member_count": 34,
  "item_count": 112
}
```

### Response fields

<ResponseField name="id" type="string">
  The unique identifier for the space.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the space.
</ResponseField>

<ResponseField name="description" type="string">
  A short description of the space's purpose or content.
</ResponseField>

<ResponseField name="slug" type="string">
  The URL-friendly slug used in the Company Brain web app.
</ResponseField>

<ResponseField name="color" type="string">
  The hex color code assigned to the space.
</ResponseField>

<ResponseField name="icon" type="string">
  The Font Awesome icon name assigned to the space.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the space was created.
</ResponseField>

<ResponseField name="member_count" type="integer">
  The number of users who are members of this space.
</ResponseField>

<ResponseField name="item_count" type="integer">
  The number of knowledge items published in this space.
</ResponseField>

## POST /v1/spaces — Create a new space

Creates a new space in your workspace. Your API key must have the **Admin** scope to use this endpoint. The authenticated user will automatically be assigned as the space owner.

<Note>
  Creating a space via the API requires an API key with the `Admin` scope. You can configure API key scopes in **Settings → API Keys** in the Company Brain dashboard.
</Note>

### Request body

<ParamField body="name" type="string" required>
  The display name of the new space. Must be between 1 and 100 characters and unique within your workspace.
</ParamField>

<ParamField body="description" type="string">
  A short description of the space's purpose. Displayed in the spaces directory and shown to users before they join.
</ParamField>

<ParamField body="color" type="string">
  A hex color code to visually identify the space in the UI (e.g. `#E11D48`). Defaults to a system-assigned color if omitted.
</ParamField>

<ParamField body="icon" type="string">
  A Font Awesome icon name to represent the space (e.g. `shield`, `code`, `chart-bar`). Defaults to `folder` if omitted.
</ParamField>

### Example request

```bash
curl -X POST "https://api.companybrain.ai/v1/spaces" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Engineering",
    "description": "Technical documentation, runbooks, and architecture decisions for the engineering team.",
    "color": "#0EA5E9",
    "icon": "code"
  }'
```

### Example response

```json
{
  "id": "sp_03ab",
  "name": "Engineering",
  "description": "Technical documentation, runbooks, and architecture decisions for the engineering team.",
  "slug": "engineering",
  "color": "#0EA5E9",
  "icon": "code",
  "created_at": "2024-04-18T12:00:00Z",
  "member_count": 1,
  "item_count": 0
}
```

## GET /v1/spaces/:id/members — List space members

Returns a list of all users who are members of the specified space, along with their role within that space.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the space whose members you want to list.
</ParamField>

### Example request

```bash
curl "https://api.companybrain.ai/v1/spaces/sp_01hx/members" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Example response

```json
{
  "data": [
    {
      "user_id": "usr_88def",
      "name": "Priya Sharma",
      "email": "priya@example.com",
      "role": "owner"
    },
    {
      "user_id": "usr_31ghi",
      "name": "Marcus Webb",
      "email": "marcus@example.com",
      "role": "editor"
    },
    {
      "user_id": "usr_55mno",
      "name": "Sofia Torres",
      "email": "sofia@example.com",
      "role": "viewer"
    }
  ]
}
```

### Response fields

<ResponseField name="data" type="array">
  An array of space member objects.

  <Expandable title="Member fields">
    <ResponseField name="user_id" type="string">
      The unique identifier of the workspace member.
    </ResponseField>
    <ResponseField name="name" type="string">
      The full display name of the member.
    </ResponseField>
    <ResponseField name="email" type="string">
      The email address of the member.
    </ResponseField>
    <ResponseField name="role" type="string">
      The member's role within this specific space. One of `owner`, `admin`, `editor`, or `viewer`.
    </ResponseField>
  </Expandable>
</ResponseField>
