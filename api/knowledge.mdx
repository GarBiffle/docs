The Knowledge API lets you programmatically manage knowledge items across your Company Brain workspace. You can create new items, update existing ones, retrieve them by ID or with filters, and delete them — making it straightforward to sync content from external systems, build automated publishing pipelines, or integrate Company Brain into your own internal tooling.

## GET /v1/knowledge — List knowledge items

Returns a paginated list of knowledge items the authenticated API key has access to. Use the query parameters below to narrow results by space, tag, author, or last-updated date.

### Query parameters

<ParamField query="space_id" type="string">
  Filter results to items belonging to a specific space. Pass the space's UUID.
</ParamField>

<ParamField query="tag" type="string">
  Filter results to items that carry a specific tag. Pass the tag slug (e.g. `onboarding`).
</ParamField>

<ParamField query="author_id" type="string">
  Filter results to items created by a specific user. Pass the user's UUID.
</ParamField>

<ParamField query="updated_after" type="string">
  Return only items updated after this timestamp. Must be an ISO 8601 datetime string (e.g. `2024-01-15T09:00:00Z`).
</ParamField>

<ParamField query="q" type="string">
  A keyword search query. Filters items whose title or body contains the provided text. For semantic AI search, use the [Search API](/api/search) instead.
</ParamField>

<ParamField query="page" type="integer" default="1">
  The page number to return. Starts at `1`.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  The number of items to return per page. Maximum is `100`.
</ParamField>

### Example request

```bash
curl "https://api.companybrain.ai/v1/knowledge?space_id=sp_01hx&tag=onboarding&per_page=2" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## GET /v1/knowledge/:id — Get a single knowledge item

Retrieves the full content and metadata for a single knowledge item by its ID. Unlike the list endpoint, this returns the complete `body` of the item rather than a preview.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the knowledge item to retrieve.
</ParamField>

### Example request

```bash
curl "https://api.companybrain.ai/v1/knowledge/ki_01abc123" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json
    "id": "usr_88def",

  "updated_at": "2024-03-05T09:11:00Z",

    "type": "manual",
```

### Response fields

<ResponseField name="id" type="string">

</ResponseField>

<ResponseField name="title" type="string">
  The title of the knowledge item.

## POST /v1/knowledge — Create a knowledge item

Creates a new knowledge item in your workspace. You must supply at minimum a `title` and a `body`. The item will be attributed to the user associated with your API key.

### Request body

<ParamField body="title" type="string" required>
  The title of the knowledge item. Must be between 1 and 500 characters.
</ParamField>

<ParamField body="body" type="string" required>
  The full body of the knowledge item in Markdown format.
</ParamField>

<ParamField body="tags" type="array of strings">
  An array of tag slugs to attach to the item (e.g. `["onboarding", "hr"]`). Tags that do not already exist will be created automatically.
</ParamField>

<ParamField body="space_ids" type="array of strings">
  An array of space IDs to publish this item to. If omitted, the item will not be added to any space.
</ParamField>

<ParamField body="visibility" type="string" default="public">
  Controls who can see the item. One of `public` (all workspace members), `space` (only members of the specified spaces), or `private` (only the author and admins).
</ParamField>

### Example request

```bash
curl -X POST "https://api.companybrain.ai/v1/knowledge" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Expense Reimbursement Policy",
    "body": "## Overview\n\nAll business expenses must be submitted within 30 days of purchase...",
    "tags": ["finance", "policy"],
    "space_ids": ["sp_01hx"],
    "visibility": "space"
  }'
```

### Example response

```json
{
  "id": "ki_03efg789",
  "title": "Expense Reimbursement Policy",
  "body": "## Overview\n\nAll business expenses must be submitted within 30 days of purchase...",

  "space_ids": ["sp_01hx"],

    "email": "priya@example.com"
  },
  "visibility": "space",
  "created_at": "2024-04-18T11:30:00Z",
  "updated_at": "2024-04-18T11:30:00Z",
  "source": {
    "type": "manual",
    "url": null
  }
}
```

## PUT /v1/knowledge/:id — Update a knowledge item

Updates an existing knowledge item. All body fields are optional — only the fields you include will be changed. Any fields you omit remain unchanged.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the knowledge item to update.
</ParamField>

### Request body

<ParamField body="title" type="string">
  A new title for the knowledge item.
</ParamField>

<ParamField body="body" type="string">
  The updated body content in Markdown format. This replaces the entire existing body.
</ParamField>

<ParamField body="tags" type="array of strings">
  A new set of tag slugs. This replaces the item's existing tags entirely — to keep existing tags, include them in the array.
</ParamField>

<ParamField body="space_ids" type="array of strings">
  A new set of space IDs. This replaces the item's existing space assignments entirely.
</ParamField>

<ParamField body="visibility" type="string">
  Update the visibility setting. One of `public`, `space`, or `private`.
</ParamField>

### Example request

```bash
curl -X PUT "https://api.companybrain.ai/v1/knowledge/ki_03efg789" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Expense Reimbursement Policy (Updated)",
    "tags": ["finance", "policy", "updated-2024"]
  }'
```

### Example response

```json
{
```

## DELETE /v1/knowledge/:id — Delete a knowledge item

Permanently deletes a knowledge item. This action cannot be undone. On success, the API returns `204 No Content` with an empty response body.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the knowledge item to delete.
</ParamField>

### Example request

```bash
curl -X DELETE "https://api.companybrain.ai/v1/knowledge/ki_03efg789" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Response

A successful deletion returns HTTP `204 No Content` with no response body. If the item does not exist or your API key does not have permission to delete it, you will receive a `404` or `403` error respectively.

<Warning>
  Deleting a knowledge item is permanent and cannot be reversed. If you need to temporarily hide an item, consider updating its `visibility` to `private` instead.
</Warning>
